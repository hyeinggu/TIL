# HTTP

웹에서 데이터를 주고받는 기본 프로토콜이며, 전송 내용이 암호화되지 않아 중간에서 가로채기·변조 위험이 크다.

## HTTPS

HTTP에 SSL/TLS를 적용해 통신을 암호화하고, 인증서를 통해 사이트의 신뢰성을 높이며 검색엔진 순위·사용자 신뢰도에 유리하다.

- SSL (Secure Sockets Layer) : 인터넷상에서 웹 브라우저와 서버 간 데이터를 암호화하여 통신을 보호하는 보안 프로토콜이다.
- TLS (Transport Layer Security) : SSL을 개선한 최신 보안 프로토콜로 더 강력한 암호화와 보안 기능을 제공하며 핸드셰이크 과정을 개선해 보안성과 통신 효율을 높였다.
- 핸드셰이크 : 웹 브라우저와 서버가 실제 데이터를 주고받기 전 서로의 신원을 확인하고 암호화 방식 및 세션 키를 교환하는 안전한 통신 수립 과정

## 요청 메시지 (Request)

1. 시작 행: HTTP 메서드(요청 내용), 요청 URL(요청을 보내는 서버 주소), HTTP 버전이 들어간다.
2. 헤더: 요청과 관련된 부가 정보가 들어간다.
3. 공백 행: 헤더의 끝을 표시하기 위해 빈 행을 넣는다.
4. 본문: 요청 내용에 따라 생략할 수 있다. 어떤 데이터를 저장하거나 수정하라는 요청의 경우 본문이 들어가고, 단순 조회 또는 삭제하라는 요청의 경우 본문이 들어가지 않는다.

## 응답 메시지 (Response)

1. 시작 행: HTTP 버전, 상태 코드와 상태 메시지(요청에 대한 응답 상태와 메시지)가 들어간다.
2. 헤더: 응답과 관련된 부가 정보가 들어간다.
3. 공백 행: 헤더의 끝을 표시하기 위해 빈 행을 넣는다.
4. 본문: 응답할 데이터가 있으면 전송한다.

## HTTP Method

- GET : 데이터를 읽거나(Read) 검색(Retrieve)할 때에 사용된다.
- POST : 새로운 리소스를 생성(create)할 때 사용된다.
- PUT : 리소스를 생성 / 업데이트하기 위해 서버로 데이터를 보내는 데 사용된다.
- DELETE : 지정된 리소스를 삭제한다.
- PATCH : PUT은 지정한 데이터를 전부 수정하고, PATCH는 정보의 일부분을 수정한다.

## 상태코드

### 범주별 분류

- 200-299 : 성공 (요청이 정상적으로 처리됨)
- 300-399 : 리디렉션 (다른 URL로 이동해야 함)
- 400-499 : 클라이언트 오류 (요청 자체에 문제가 있음 예: 잘못된 문법, 권한 부족 등)
- 500-599 : 서버 오류 (서버에서 요청 처리 중 오류 발생)

### 상태코드 설명

- 200(성공): 성공적으로 요청 처리
- 201(생성 성공): 새 리소스 생성됨
- 400(잘못된 요청): 요청이 잘못됨 (파라미터, JSON 형식 다시 확인)
- 401, 403(인증/권한): 인증 필요/권한 없음 (토큰 유효성 확인/접근 권한 확인 필요)
- 404(찾을 수 없음): URL이 존재하지 않음 (경로 확인 또는 오타 수정)
- 500(서버 오류): 서버 내부 오류 (서버 로그 확인, 개발자 대응 필요)
