# Git Flow

## Git Flow란?

- **Git**을 사용할 때 **프로젝트의 브랜치를 체계적으로 관리하는 방법**
- 기능 개발, 테스트, 배포 등을 **각각 다른 브랜치에서 작업**하도록 구조 나누어 관리
- 여러 개발자가 동시에 작업해도 **코드 충동을 줄이고 효율적으로 협업 가능**
- 보통 **GitHub**에서 팀 프로젝트를 진행할 때 사용

## 주요 브랜치 종류

- **Main(Master)**: 최종 완성된 안정적인 코드
- **Develop**: 개발 중인 코드
- **Feature**: 새로운 기능을 개발할 때 사용하는 브랜치
- **Release**: 배포 준비 단계에서 사용하는 브랜치
- **Hotfix**: 배포 후 긴급한 오류를 수정할 때 사용하는 브랜치
