# Git

## Git이란?

코드의 변경사항을 계속 감시하고 기록하는 '**버전 관리 시스템**'

## Git의 핵심 기능

-**버전 관리**: 프로젝트 내 문서의 수정 이력을 체계적으로 관리하여 특정 시점으로 되돌리거나, 서비스 버전과 개발 버전을 별도로 운영하는 등 프로젝트 관리에 편리함 제공

-**협업 체계**: 여러 사람이 하나의 프로젝트를 함께 수행할 때, 각자 독립적인 개발 버전을 관리하며 협업할 수 있음. 원격 저장소를 통해 USB나 이메일 없이도 코드를 주고받고 합치는 작업 용이

## Git 사용절차 (Add, Commit, Push)

-**Add**: 원격으로 보낼 **코드 선택**

-**Comiit**: 코드 **변경 이유를 설명**하는 과정

-**Push**: 로컬 코드를 **원격으로 동기화**

-커밋 메시지는 **미래의 나를 위해** 상세히 작성해야 함

## Git 사용 시 주의사항

-**경로에 한글 금지**: **에러 발생**의 주요 원인

-**용량 제한**: 50MB 초과 시 경고, 100MB 초과 시 에러 발생

-**대용량 파일**: Google Drive, Dropbox 등 **별도 경로 활용**

-**토큰 사용**: GitHub 로그인 시 **패스워드 대신 토큰** 사용
